# GitHub Reference

Repository remote target: `https://github.com/8dazo/cap`

## Expected delivery flow

1. Capture the work in a GitHub issue.
2. Implement on a feature branch.
3. Open a pull request that references the issue.
4. Merge the pull request into `main`.

## Guardrails

- Do not work directly on `main` for feature changes.
- Keep issue and PR titles aligned with the actual scope.
- Prefer one clear issue and one clear PR per coherent change.
- If the local repo is not connected yet, set the `origin` remote to the repository above before pushing.
